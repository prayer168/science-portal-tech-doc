05 · Icon Map

MATERIAL_ICON_MAP

由於 Firestore 資料優先，直接改 DEFAULT_DATA 的 icon 無效。所有 icon 修正透過全域對照表 MATERIAL_ICON_MAP + applyIconMap() 實作。

index.html — 搜尋 MATERIAL_ICON_MAP
const MATERIAL_ICON_MAP = {
  // 休閒動動腦
  '加特林射擊場': '🎯',  '深海水母': '🌊',  'Q版俄羅斯方塊': '🧩',
  'QR Code 生成器': '🔳',
  // 三年級
  '雨量筒水位模擬1': '🌧️',  '雨量筒水位模擬2': '🌧️',  '雨量筒水位模擬3': '🌧️',
  '虹吸現象大解密': '💧',
  // 四年級
  '月相變化': '🌙',  '植物蒸散作用': '🌫️',  '溫度與物質變化': '⚗️',
  // 五年級
  '傅科擺': '🌍',  '自然知識大冒險': '🗺️',  '月相盈虧互動遊戲': '🌙',
  // 教學應用
  '教室互動儀表板': '📊',  '智慧錄音': '🎙️',  '網站使用手冊': '📋',
  // ... 完整列表見 index.html
};

function applyIconMap(arr) {
  arr.forEach(d => { if (MATERIAL_ICON_MAP[d.name]) d.icon = MATERIAL_ICON_MAP[d.name]; });
  return arr;
}
🚨

定義順序至關重要：applyIconMap 函式必須定義在 loadDataLocal() 被呼叫之前，否則會出現「函式未定義」錯誤。

💡

新增 icon 對應：在 MATERIAL_ICON_MAP 加入 '教材完整名稱': '🎯'，名稱需與 Firestore 的 name 欄位完全一致（包含空格、全半形）。

07 · Deployment

部署流程

方法 A：Git CLI（推薦）

Terminal / PowerShell
# 取得最新版本
git clone https://github.com/prayer168/science-portal.git
cd science-portal

# 修改 index.html 後推送
git add index.html
git commit -m "修改說明"
git push origin main

# → GitHub Pages 約 1 分鐘後生效

方法 B：GitHub Contents API（無 git 環境時）

Python 3
import json, urllib.request, base64

TOKEN = "ghp_xxxxxxxx"
URL   = "https://api.github.com/repos/prayer168/science-portal/contents/index.html"

# 1. 取得目前 SHA（必要，否則推送失敗）
req = urllib.request.Request(URL, headers={"Authorization": f"token {TOKEN}"})
sha = json.loads(urllib.request.urlopen(req).read())['sha']

# 2. 讀取修改後的檔案並 Base64 編碼
with open('index.html', 'rb') as f:
    encoded = base64.b64encode(f.read()).decode()

# 3. 推送
data = json.dumps({"message": "commit message", "content": encoded, "sha": sha}).encode()
req  = urllib.request.Request(URL, data=data, method='PUT')
req.add_header('Authorization', f'token {TOKEN}')
req.add_header('Content-Type', 'application/json')
urllib.request.urlopen(req)
⚠️

GitHub Token 注意：Token 每次使用需重新提供。建議到 github.com/settings/tokens/new 產生，勾選 repo 權限即可，有效期設 7 天。

08 · Pitfalls

重要陷阱（踩過的坑）

坑 1 改 DEFAULT_DATA 完全沒有用
問題：修改程式碼裡的 icon、名稱，前台完全看不到變化。
原因：Firestore 資料載入後會蓋過 DEFAULT_DATA，localStorage 快取也優先。
解法：改 MATERIAL_ICON_MAP，並確保 applyIconMap 同時掛在 loadDataLocal() 和 _firestoreDataCallback() 兩條路徑上。
坑 2 applyIconMap 定義順序錯誤
問題：applyIconMap is not defined 錯誤。
原因：函式在 loadDataLocal() 呼叫之後才定義。
解法：MATERIAL_ICON_MAP + applyIconMap 必須定義在 loadDataLocal() 前面。
坑 3 教材名稱必須完全一致
問題：「月相變化」和「月相盈虧互動遊戲」是兩筆不同教材，ICON_MAP 只對其中一個。
解法：ICON_MAP 的 key 必須與 Firestore 裡的 name 欄位字元完全相同，包含空格、全半形。
坑 4 類別改名要同步更新 cat 欄位
問題：類別改名後，舊類別下的教材全部消失。
原因：教材的 cat 欄位還是舊名稱，找不到對應類別。
解法： DATA.forEach(d => { if (d.cat === oldKey) d.cat = newName; });
坑 5 拖曳排序的 index 計算錯誤
問題：拖曳後教材跑到錯誤位置。
原因：splice 移除元素後，目標 index 若在來源之後需要 -1。
解法：const adjusted = tgtIdx > srcIdx ? tgtIdx - 1 : tgtIdx;
坑 6 sidebar 顯示在右邊
問題：HTML 結構 main-col 在前、sidebar 在後，grid 欄位順序影響顯示位置。
解法：grid: 236px 1fr + .sidebar { order: -1 } 讓目錄顯示在左側。
坑 7 favicon 用 href 路徑載入失敗
問題：直接 href="favicon.png" 在某些情境下載入失敗或有白色背景。
解法：將圖片轉成 base64，以 data URI 嵌入 <link> 標籤。
坑 8 GitHub Contents API 大小限制
問題：超過約 100KB 的檔案用 Contents API 推送可能失敗。
解法：改用 git CLI 推送，或分段壓縮。
